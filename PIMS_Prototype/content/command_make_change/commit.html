Official Git Documentation - Commit

Commit

DETAILS

Performing a 'commit' is the process of applying staged (added) changes to the current branch.

git commit -m "yourcomments"

Applies staged changes to the current branch with the comment of "yourcomments".

git commit --ammend --no-edit

--ammend will apply newly staged changes to the most recent commit. More accurately, it replaces the 'tip' of the current branch with a new commit.

--no-edit will re-use the most recent commit message
