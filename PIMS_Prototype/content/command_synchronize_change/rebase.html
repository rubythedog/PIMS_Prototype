Rebase

DETAILS

Performing a "rebase" is the process of re-applying a current branches commits on top of another 'tip'. Normally done when the is 'ahead' of the local branch (aka ). Should be done to prepare for a .

git rebase <upstream_branch> <current_branch>

Moves the 'base' of the <current_branch> to the 'tip' of <upstream_branch> branch.

This is the transactional equivalent of:

$ git checkout <current_branch> (1)
$ git rebase <upstream_branch>  (2)

  1. checkout branch

  2. rebase to

EXAMPLE

Assume the following history exists:

          A---B---C feature_branch
         /
    D---E---F---G master

From this point, the result of the following command:

git rebase master feature_branch

would be:

                  A'--B'--C' feature_branch
                 /
    D---E---F---G master
