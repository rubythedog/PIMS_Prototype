Official Git Documentation - Push

Push

DETAILS

Performing a 'push' is the process of updating a 'remote branch' with anything missing from a 'local branch'.

git push -u origin <feature_branch>

Updates <origin/feature_branch> with any the missing commits from <feature_branch>. If <origin/feature_branch> does not exist, it is created.

'-u' adds upstream tracking reference - in case 'git pull' without arguments is used.

git push -f origin <feature_branch>

Same as above. However, the '-f' forces the update to <origin/feature_branch> even though it is not an ancestor of <feature_branch>. This can effectively re-write history of <origin/feature_branch>. Use with caution (and the --dry-run option).
