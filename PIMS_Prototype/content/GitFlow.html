GitFlow

Overview

A Git Workflow dictates what kind of branches to set up and how to merge them together. One of the most popular branching and merging workflow models is GitFlow. At a glance, this is the GitFlow Workflow.

IMPORTANT: There are two main branches with an infinite lifetime:

  • master
  • develop
Basic Workflow
  1. A develop branch is created from master
  2. A release branch is created from develop
  3. A feature branch is created from develop
  4. When a feature is complete it is merged into the develop branch
  5. When it is time to make a release, a release branch is created off of develop
  6. When the release branch is done it is merged into develop and master
  7. If an issue in master is detected a hotfix branch is created from master
  8. Once the hotfix is complete it is merged to both develop and master
