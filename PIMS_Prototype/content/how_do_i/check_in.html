Check-in Code (aka 'Commit')

This page lists the steps to check-in code (commit( code using the normal (happy path) workflow. i.e. no merge issues.

Steps

Action Command
Switch to <base_branch>git checkout <base_branch>
Get latest of <base_branch>git pull
Create local <feature_branch>git checkout -b <feature_branch> <base_branch>
*****Edit files / do some work*****-------------
Stage changes to <feature_branch>git add .
Commit changes to <feature_branch>git commit -m "yourcomments"
Push changes to remotegit push -u origin <feature_branch>
Initiate (remote) merge to <base_branch>merge branches (via web browser)
Complete (remote) merge to <base_branch>complete pull request (via web browser)
Switch to 'master' (or any other branch)git checkout <master>
Delete local <feature_branch>git branch -d <feature_branch>
Delete (aka 'Prune') local orphaned branchesprune branches
