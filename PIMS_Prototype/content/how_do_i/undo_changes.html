Undo Changes

If you want to undo/undelete/restore/discard changes done to your branch, follow the steps below. The steps to follow depend on the state of the changes.

I Want To:
  • Undelete Files / Folders - that were accidentally removed (git status lists as 'red')
  • Undo Changes - to unstaged files/folders (git status lists as 'red')
  • Undo Changes - to staged files/folders (git status lists as 'green')
  • Change Staged (files / folders) to Unstaged - of files / folders (from 'green' -> 'red')
  • Undo a Commit - made to a branch
  • Make Everything The Same - make local branch exactly match remote (if you don't know what to choose)

Note: This assumes the branch to perform 'undo' actions on is the 'current' branch. Use git checkout <feature_branch>

Undelete Files / Folders

Choose this when you want to recover files or folders that have been accidentally deleted (but have not been staged).

Perform this action on red files/folders (from git status).

Action Command
Undelete files / foldersgit checkout -- [PATHSPEC]

Afterwards, the files / folders will not appear using git status

Undo Changes (unstaged)

Choose this when you want to undo changes to unstaged files / folders.

Perform this action on red files/folders (from git status).

Action Command
Undo changesgit checkout -- [PATHSPEC]

Afterwards, the files / folders will not appear using git status

Undo Changes (staged)

Choose this when you want to undo changes to staged files / folders.

Perform this action on green files/folders (from git status).

Action Command
Unstage files / foldersgit reset -- [PATHSPEC]
Undo changesgit checkout -- [PATHSPEC]

Afterwards, the files / folders will not appear using git status

Change Staged (files / folders) to Unstaged

Choose this when you want to remove files/folders from being eligible to be commited.

Perform this action on green files/folders (from git status).

Action Command
Unstage files / foldersgit reset -- [PATHSPEC]

Afterwards, the files / folders will appear as red using git status

Undo a Commit (made to a branch)

Action Command
Undo (revert) a commit made to a branchgit revert --no-edit [SHA-1]

Everything

WARNING!! Choosing this option will delete all changes in your folder that have made!!!

Action Command
Switch to <feature_branch>git checkout <feature_branch>
Undo changes done to <feature_branch>git reset --hard <Origin/feature_branch>
Delete any untracked files & foldersgit clean -d -f
