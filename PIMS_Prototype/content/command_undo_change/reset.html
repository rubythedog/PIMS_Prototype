Official Git Documentation - Reset

Reset (undo change to index)

DETAILS

Performing a 'reset' is the process of returning a branch to a state before any merges.

If you are 'resetting' from origin, you are undoing all local commits - effectively if you delete the working tree and get a new copy.

If you are 'resetting' a file / folder, you are moving it from being staged (to be eligible to be comitted) to unstaged.

git reset -- [PATHSPEC]

Removes from being staged.

git reset --hard <Origin/feature_branch>

Discards any commits on the current branch in your working tree and makes the current branch identical to <Origin/feature_branch>.
